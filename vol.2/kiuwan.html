[T-리포트] 모바일 앱 보안을 위한 5가지 팁

 

모바일 앱이 인기 있는 이유 중 하나는 사용자들이 스마트폰 하나로 원하는 정보를 다 얻을 수 있기 때문입니다. 사용자들은 민감한 개인 데이터를 스마트폰에서 거리낌 없이 사용합니다. 그리고 스마트폰에 설치된 앱이야말로 개인정보를 탈취하는 가장 큰 관문이 되고 있습니다.

불과 몇 년 전 IBM 보고서에 따르면 모바일 앱 보안에 관한 예산이 없는 기업이 50%에 달했습니다. 2019년 상반기에만 약 41억 건의 데이터 유출/침해 사고가 발생한 것도 매우 우려스러운 상황입니다. 최근에는 75%의 앱이 사용자를 사기나 신원 도용 위험에 빠지게 할 민감한 데이터를 유출했다는 조사도 있었어요.

사고가 자꾸 증가하면서 사용자의 개인정보를 지킬 책임이 모바일 앱 개발자에게 향하고 있죠. 앱 사용자를 데이터 침해 사고로부터 보호하려면 어떤 방법을 써야 하는가에 관하여, 몇 가지 중요한 포인트를 짚어 알려드립니다.

 

 

1. 앱 소스코드 보안은 처음부터!

 

앱 보안에 신경을 쓰고 싶으신가요? 앱을 배포한 후에 시작하면 늦습니다. 보안은 개발 프로세스의 일부여야 합니다. 보안이 최우선 사항이 되도록 하고, 개발 작업 중의 오류나 테스트가 미비하여 발생한 취약점으로부터 코드를 보호해야 해요.
특히 모바일앱의 코드를 보호하기 위해선 API 암호화와 결합한 코드 난독화 등을 통해 암호화된 상태로 유지해야 합니다. 소스코드를 스캔하면서 앱의 보안 사항이 사용자 경험과 앱의 퍼포먼스에 영향을 주지 않는지 늘 체크하세요.

 

2. 로그인 과정에 보안 시스템 추가하기

 

버라이즌(Verizon)이 2019년에 발표한 ‘데이터 보안 침해 조사 보고서’에 따르면 데이터 유출/침해 사고의 약 80%가 ‘개인정보 암호화’가 취약한 데서 발생합니다. 따라서 로그인할 때 정보가 탈취되지 않도록 2단계 인증방법인 2FA(two-factor authentication)를 추가할 것을 권하는 추세입니다.
또한 앱 사용자에게 복잡한 비밀번호를 설정하고 정기적으로 업데이트하게 권장하는 것은 가장 기본적인 방법입니다. 이를 위해 강력한 영숫자 비밀번호만 등록할 수 있게 앱을 설계하는 것도 필요하겠죠?

 

3. API 보안

 

앱 개발은 API에 크게 의존하기 때문에 API 보안 전략을 견고하게 세워야 합니다. 네트워크에 접근하는 사람이면 누구나 인증을 거치게끔 하세요. 그리고 API 게이트웨이도 꼭 두어야 합니다. 네트워크 보안을 위해 VPN 등을 사용할 수도 있을 텐데요, 만약 어렵다면 보안 조치로 컨테이너화(containerization)를 고려해보세요. 그리고 API 보안스택에는 권한부여, 식별 및 인증에 대한 조치가 있어야 합니다.
(* 컨테이너화란 호스트 OS 상에 논리적인 구획인 컨테이너를 만들고, 앱 실행에 필요한 라이브러리나 애플리케이션 등을 하나로 모아 마치 별도 서버인 것처럼 사용할 수 있게 만든 것을 말합니다.)

 

4. 데이터 암호화 정책 구현

 

앱에서 오가는 모든 데이터가 암호화됐는지 꼭 확인하세요. 이걸 잘해놓는다면 만약 해커가 보안을 통과하더라도 민감한 정보에는 액세스할 수 없게 됩니다. 민감한 데이터를 모바일 기기나 서버에 가급적 저장하지 말고, 꼭 해야 한다면 암호화된 형식으로 최소한의 데이터 저장소만을 두세요. iOS 키체인이 암호화된 데이터 저장소를 갖고 있는 좋은 예입니다.

 

5. 취약성 테스트 진행

 

해커가 오늘 당장 모바일 앱을 공격한다면 잘 버틸 수 있을까요? 해커가 공격할 수 있다는 위험성에 대해 머리로는 인지하지만, 아주 심각하게 받아들이는 경우는 드뭅니다. 현실성이 없게 느껴지기 때문이죠. 그러나 ‘운’에 앱의 운명을 맡겨서는 안 될 일! 정기적으로 꼭 취약성 테스트를 수행할 것을 권합니다. 취약성 테스트로 앱 인프라의 모든 보안상 허점을 식별할 수 있고, 사태가 벌어지기 전에 취약점을 개선할 수 있게 도와줄 거예요.

키우완(Kiuwan)을 비롯해 애플리케이션의 취약성을 스캔할 수 있는 좋은 도구들이 있으니 꼭 이용해보세요. 특히 키우완은 오픈소스 코드의 보안 취약점을 분석하여 개선할 수 있게 도와주는 SAST(Static Application Security Testing) 도구입니다. SCA 분석을 활용하여 DevOps 환경 내에서 원활하게 확장 가능한 빠른 플랫폼을 통해 보안 위험으로부터 애플리케이션을 보호할 수 있도록 해줍니다. 이렇게 적절한 SAST 도구를 사용하면 위에서 언급한 앱 보안에 관한 많은 것을 놓치지 않고 진행할 수 있어요.

이상으로 모바일 앱 보안이 왜 중요한지, 앱 보안을 위해서 어떤 방법을 써야 하는지 알아보았습니다! 더 많은 정보를 원하신다면 Kiuwan Blog 원문에서 확인하실 수 있습니다.
